# V1.1更新思路
### 类型的增加与修改
##### 葫芦娃类的重写
1. 对于enum不是很理解用法，直接将葫芦娃分成了CalabashGuy和CalabashBrothers两个类
2. CalabashGuy类继承生物体类，并且定义了单个的葫芦娃的属性，未来考虑依据葫芦娃的等级来设定技能
3. CalabashBrothers通过组合将七个葫芦娃封装在里面，同时设定私有变量num=7，来保护葫芦娃的数量为7

##### Good和Bad类
1. 将阵营分为两拨，一方是好的，一方是坏的。
2. 每个阵营中设定了阵营所拥有的生物体、以及各自的阵型变量。
3. 重写了selectFormation函数，使它看起来跟符合现实。
```javascript
public class Good {
	Grandpa grandpa=new Grandpa();
	CalabashBrothers calabash=new CalabashBrothers();
	Formation goodFormation;
	...
}
```
```javascript
public class Bad {
	Sneak sneak;
	Sorpio sorpio;
	ArrayList<Lackeys> LackeysList;
	Formation badFormation;
	...
}
```

##### Formation类
1. 将阵型类重写，改为了一个接口，接口中包含了对好阵营和坏阵营的排列方法。
2. 将每个阵型从一个类中的一个一个方法，改写成了一个一个类，并且每个阵型拥有自己的名字，将耦合度降低了（是这么说的吧？）

##### Position类
1. 增加了Position类，生物体作为成员变量，另外设定了自己的x、y坐标，但是可能还没有用。
2. 将位置的判断方法写入Position类中

##### 重写Space类 修改为Battlefield类
1. 原来的空间是用String二维数组实现的，将String二维数组改为Position的二位数组

### 亮点之处
1. 将每个阵型改写成了方法，降低了耦合度。
2. 增加了Position类，Good类，Bad类，更好地对现实世界模拟。
3. 对于喽变量，利用了ArrayList容器，删去了原来的数组，使得喽的数量可以更灵活了。

# V1.0
# 抽象类
##### Main
作为一个创世界的函数，声明了一个二维空间，由这个二维空间来展现葫芦娃和蝎子精之间的对抗。
##### 生物体
1.生物体中设置了名字，坐标等属性，以及设置坐标的方法。
2.利用继承的思想，将葫芦娃、蝎子精等作为生物体的子类，这样这些生物体都可以使用设置坐标setXY的方法了。
##### 空间
1.在空间中将了所有的生物体类实体化。
2.用setPace()方法，将设置各类生物体在空间中的位置。
3.用printSapce()方法，将空间的整个样貌打印出来。
##### 阵型
阵型这个类中，将每个阵型写成了方法，阵型的实现用到了生物体的setXY方法。其中长蛇阵牵扯到了函数的重载。

# 反思
1.对面向对象的思想了解了一些，但还是很难适应这样的思维。
2.要将整个世界抽象成一个一个类，所以我觉得这个世界有的就是一个二维空间，各种各样的生物体，以及各种各样的阵型。
3.对于生物体，没有想到要将他们集合成正义的一方或邪恶的一方，我看了别的同学的代码，很多人都有这么做，在下次修改中应该加入这个思想。
4.对于阵型来说，我将阵型设置成了一个一个的方法，可能把阵型写成一个一个的类会更好点，这样生物体选择阵型的时候就可以直接调用类，而不是调用方法，这个地方我觉得我需要重新想一下。
